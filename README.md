## PikPak 文件分享的两种方式

PikPak 目前有两种主流的文件分享方式，理清楚了用起来才顺手。

### 方式一：分享链接（常规方式）

这是最基础的操作，跟大多数网盘逻辑一样：

1. 打开 PikPak App，找到要分享的文件或文件夹
2. 点击右侧的「三个点」菜单
3. 选择「分享」
4. 点击「创建分享」
5. 生成链接后，点击「复制链接」

复制出来的是一段 `https://mypikpak.com/s/...` 格式的链接，发给对方就行。对方打开链接，可以预览文件，点击「保存到云盘」一键存入自己的 PikPak 账号。

整个过程不需要对方下载到本地，文件直接在云端完成转移，速度很快。

### 方式二：分享代码（Share Code，PikPak 特色功能）

分享代码是 PikPak 的一个独特功能，本质是「更短的分享链接替代品」。

相比一串长长的 URL，分享代码可以设置成几个字母+数字的组合，比如 `abc123`，更方便口头传递或者在聊天时输入。

**创建分享代码的步骤：**

1. 先通过上面的方式创建一个分享链接
2. 打开 👉 [PikPak 网页版「我的分享」页面](https://mypikpak.com/drive/shared)
3. 找到刚才创建的分享条目，点击右侧「查看分享详情」
4. 在弹窗里点击「添加分享代码」
5. 自定义一个由数字、字母、符号组成的代码
6. 点击保存

**接收方如何使用分享代码：**

- 在 PikPak App 首页，找到「输入分享代码」的卡片，直接输入代码
- 或者在「添加链接」界面输入代码
- 或者在搜索框输入代码，按提示操作

> ⚠️ 注意：分享代码目前主要在 **Android App** 中支持，iOS 和网页版可能存在差异，建议以实际操作为准。

## PikPak 保险库的加密分享（高阶功能）

如果你要分享的是比较私密的文件，PikPak 还有一个「保险库加密分享」功能。

这个功能的安全逻辑是：每次分享会生成**独立的分享链接 + 分享密钥**，两者缺一不可。也就是说，就算有人截获了你的分享链接，没有密钥也解密不了内容——包括 PikPak 自己也看不了。

还有几个实用的控制选项：

- **设置有效期**：到期后链接自动失效，不用手动去撤销
- **随时删除分享**：删除后，即使对方还有链接，也会立即失效
- **分离模式**：链接和密钥分开发送，安全性更高

适合用来分享合同、证件扫描件、私人照片这类比较敏感的内容。

## 收到分享之后，怎么快速保存？

很多人知道怎么发，不太清楚怎么收。简单说三种场景：

**场景一：收到的是分享链接**
直接在浏览器或 App 内打开链接，进入预览页面，点「保存到云盘」，文件就进你自己的 PikPak 了。整个过程是云端到云端，几乎不消耗本地流量。

**场景二：收到的是分享代码**
打开 PikPak App，在首页找到分享代码输入框，输入代码，就能看到对方分享的文件，然后保存即可。

**场景三：想把 PikPak 里的文件转存到其他网盘**
这个用 PikPak 的官方功能没法直接做，但可以借助 Rclone、CloudSlinker 等第三方工具，把 PikPak 的文件导出到 Google Drive、OneDrive 等平台。不在本文展开，有需要可以另查。

另外，免费版存储空间只有 **6GB**，分享给对方之后，对方如果空间不够，根本存不下大文件。

## 关于 PikPak 分享的常见问题

**Q：分享的文件有数量限制吗？**  
A：PikPak 对分享数量没有硬性说明，但会员和非会员在具体操作体验上可能有差异。

**Q：分享链接会不会失效？**  
A：如果你没有设置有效期，链接默认是长期有效的。但如果文件被 PikPak 审核屏蔽（违反内容政策），链接会失效。

**Q：分享出去的文件，对方修改会影响我的原文件吗？**  
A：不会。对方只能查看或保存到自己的云盘，无法修改你的原始文件。

**Q：免费用户可以使用文件分享功能吗？**  
A：可以，分享功能本身免费版也能用。但免费版存储只有 6GB，收到分享的文件如果超出空间就存不下了。

**Q：iOS 用户可以创建分享吗？**  
A：iOS 用户可以正常分享文件，但分享代码功能目前主要在 Android 端支持。部分功能（如添加磁力链接）在 iOS 因为苹果政策限制需要通过网页版或 Telegram Bot 完成。
